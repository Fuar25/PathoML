# GML 数据集概况

## 数据层级

本项目中的样本按以下层级组织：

```text
类别 / 病理号 / 组织块 / 染色切片
```

- `类别`: 诊断类别，包括 `MALT` 和 `Reactive`
- `病理号`: 病例编号，例如 `B2018-14321`
- `组织块`: 同一病理号下的组织编号，例如 `A`、`B`、`C`
- `染色切片`: 同一组织块下的不同染色切片，例如 `HE`、`CD20`、`CD3`、`CD21`、`CK-pan`、`Ki-67`

本项目以“组织块”作为主要样本单位。组织块标识由病理号和组织编号组成：

```text
组织块标识 = 病理号 + 组织编号
```

例如：

```text
B2018-14321 + C = B2018-14321C
```

因此，一个病理号可以对应一个或多个组织块。

## 下游数据口径

当前下游任务使用组织块作为样本单位，并要求每个组织块同时具备以下三种染色的 GigaPath patch-level 特征：

```text
HE
CD20
CD3
```

因此，下游有效组织块数量为：

```text
组织块: 264
  MALT: 133
  Reactive: 131
```

这里的统计单位是组织块，不是病理号。

## 标签文件

标签文件位于：

```text
/mnt/5T/GML/Reg/labels.csv
```

字段：

```text
patient_id,label
```

标签以病理号为单位提供。对于同一个病理号下的多个组织块，使用相同的病理号级标签。
