# PathoML/scripts

## 1. Purpose
**Minimal use case examples** for running experiments with the core library. It's going to tell user about how to run the core functions or core library In the quickest and most concise way. Whenever the core library is updated, check if the examples are still working and update them if necessary.

## 2. Rules
- It is the direct user manual for the core library. It should be as concise as possible, and only include the necessary code to run the core functions. It should also be updated with the latest version of the core library.
